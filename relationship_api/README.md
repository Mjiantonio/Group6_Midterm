RegisterParticipant.php - This is where we create a relationship between participant and a specific event. It takes event_id and participant_id and inserts them into the registration. Then it waits for them, if one of the IDs are forgotten, the code stops and tells that the data is "Incomplete". If it was successfully created the relationship, it will tell that "Relationship Created" and if a error happens it will tell "Unable to create relationship".

EventStat.php - This is where we show the relationships. It calls getParticipantsPerEvent and grabs every event it uses JOIN and GROUP BY to count how many people are in to it in the registrations table, and puts them into a clean list. When this is called, it give the list of event names and their total headcounts. If there are no sign-ups, it will say "No relationship found".
